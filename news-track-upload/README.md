# India News Trending Scraper

Playwright-based scraper for collecting trending or most-read headlines from:

- Times of India
- NDTV
- India Today

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install chromium
```

## Usage

Scrape all supported sites:

```bash
python scrape_india_news_trending.py --site all --out_csv india_trending.csv --out_json india_trending.json
```

Scrape one site:

```bash
python scrape_india_news_trending.py --site toi
```

Use installed Chrome instead of bundled Chromium:

```bash
python scrape_india_news_trending.py --site all --channel chrome
```

Run headless:

```bash
python scrape_india_news_trending.py --site all --headless
```

Generate a single self-contained HTML dashboard:

```bash
python scrape_india_news_trending.py --site all --headless --out_html web/index.html
```

## Output

Each row contains:

- `source`
- `section`
- `rank`
- `headline`
- `link`
- `collected_at_iso`

## Hosting

The repo includes:

- `web/dashboard.html`: CSV-backed dashboard
- `web/index.html`: self-contained dashboard generated by the scraper

For Vercel or Netlify, the simplest option is to host `web/index.html` as a static site.

## Refresh Every 2 Hours

A GitHub Actions workflow is included at `.github/workflows/refresh-dashboard.yml`.

It:

- runs every 2 hours
- scrapes the latest data
- writes `web/india_trending.csv`
- writes `web/india_trending.json`
- writes `web/index.html`
- commits the refreshed files back to the repo

That means Vercel or Netlify only needs to serve the static `web/` output.

## Notes

These sites change their markup often. The scraper targets explicit trending and most-read pages for each publisher and filters for article-style URLs, but you should still expect to refresh patterns over time.
